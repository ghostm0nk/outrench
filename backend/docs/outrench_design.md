# Outrench — Product Design Notes

## Station / Terminal
The autonomous log view. Hour buttons (1hr, 2hr, 5hr, 24hr) trigger the process for the day.
Session results show the same things but with linked cards. From the terminal, the user can open links and go directly to posts from the session results.

## Channels
At the top: accounts and their credentials (overlay appears with info).
This is where the daily summary shows up — what Spirit did throughout the hour/day, what to post, and what media it needs to create a post.
Pressing Auto: posts words-only (no media required).

## Analytics
Engagements Spirit did through each day. Account follower/engagement growth in a week.

## Directory
What the user wanted to do with the profiles, what the business is. Milestones.

## Notes
A Todo list from the user to the AI. Spirit runs on the Station's terminal autonomously.
The message input is for interrupting the AI or changing direction mid-session.

## Profile
Authentication, delete account, billing, and log out.

## Onboarding
Start with personal and brand account preference first — so it doesn't get confusing later.
If the user wants to set up the brand account, they go through questionnaires again.
That's what the Directory is for.

---

## Account Types
- **Personal account** — builds the founder's brand and visibility
- **Product account** — official product announcements and brand awareness

## Modes
- **Growth mode** — find and engage the target audience (likes, follows, comments)
- **Content mode** — draft and post original content in the founder's voice
